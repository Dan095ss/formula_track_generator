# Генератор графика смен 2/2 — дизайн

**Дата:** 2026-06-03
**Статус:** утверждён к реализации
**Тип:** CLI-инструмент (терминал-only, без веб-интерфейса)

## Цель

Демонстрационный CLI на Python, генерирующий график рабочих смен по схеме «2 через 2»
для команды аналитиков, разделённой на два региона (Запад / Восток). Главная
особенность — корректное **наследование фазы цикла от месяца к месяцу**, плюс набор
дополнительных правил (ночные смены, отпуска, запросы выходных, покрытие).

## Терминология / обозначения

| Символ | Значение |
|--------|----------|
| `Д` (DAY) | дневная смена |
| `Н` (NIGHT) | ночная смена |
| выходной (OFF) | нерабочий день |
| `О` (VACATION) | отпуск |
| `Вх` | запрос выходного (на входе; в графике становится OFF) |

Регионы: **Запад** (Europe/Americas) и **Восток** (Asia/Pacific).

## Базовая логика 2/2 и наследование

Каждый аналитик крутится по непрерывному 4-дневному циклу `[Д, Д, выходной, выходной]`.
Положение в цикле на 1-е число месяца задаётся фазой `offset ∈ {0,1,2,3}`:

```
cell(offset, day_index) = DAY  если (offset + day_index) % 4 ∈ {0, 1}
                          OFF  иначе
```

Наследование месяца — сдвиг фазы:

```
offset_след_месяца = (offset_тек + n_days_тек) % 4
```

Это автоматически даёт три требуемых правила стыка месяцев:

1. Последние 2 дня выходные → первые 2 дня следующего месяца рабочие.
2. Предпоследний день выходной, последний рабочий → первый день следующего месяца рабочий.
3. Последние 2 дня рабочие → первые 2 дня следующего месяца выходные.

Следствия, выполняемые циклом «бесплатно»:
- между блоками смен всегда ≥1 выходной;
- не бывает >2 рабочих подряд (до наложения прочих правил).

## Дополнительные правила

- **Ночные (Н):** только у аналитиков с `allows_night=True`. Ставятся «по потребности
  покрытия»: ночь региона = день соседнего региона (таймзонная логика —
  ночь Запада ≈ день Востока и наоборот). Если тёмное окно региона не закрыто дневной
  сменой соседа, рабочий день кандидата с флагом переводится `DAY→NIGHT`.
  **Перед ночной всегда выходной** (день перед `NIGHT` обязан быть `OFF`/`VACATION`).
  Запад и Восток ставят ночи в противофазе.
- **Отпуск (О):** дни из `vacation` становятся `VACATION` и **учитываются сквозным образом**:
  - не считаются рабочей сменой (исключены из `shift_count`, влияют на «среднее ~15»);
  - не закрывают покрытие региона (день в отпуске ≠ работающий человек);
  - не могут стать ночными (ночной проход их пропускает);
  - сохраняют правило «не более 2 рабочих подряд при наличии отпуска» — обеспечивается
    циклом 2/2 + тем, что отпуск только укорачивает рабочие блоки;
  - очищаются при наследовании месяца (разовые, как и Вх).
- **Запрос выходного (Вх):** дни из `day_off_requests` — **жёсткое** правило, всегда
  переводятся в `OFF`. Может приводить к недобору покрытия — это допустимо, но валидатор
  такой день подсвечивает.
- **Среднее ~15 смен/мес:** мягкое правило, допустимый диапазон 12..18. При 2/2 на 30-дневном
  месяце естественно выходит ~15.
- **Минимум 2 человека в смене:** жёсткое правило, действует только если число аналитиков
  чётное и >2. Достигается шахматной раскладкой фаз.

## Архитектура

Пакет `shift_scheduler/` — чистое ядро без I/O + тонкий CLI:

```
shift_scheduler/
├── __init__.py
├── model.py        # Analyst, Region, ShiftType, MonthSchedule
├── roster.py       # встроенный демо-ростер (детерминированный)
├── generator.py    # generate(roster, year, month) -> MonthSchedule; inherit_offsets()
├── validator.py    # validate(schedule, roster) -> list[Violation]
├── render.py       # терминальный вывод: таблица-календарь + отчёт
├── cli.py          # argparse + UTF-8 stdout
└── tests/
    ├── __init__.py
    ├── test_generator.py
    ├── test_inheritance.py
    ├── test_validator.py
    └── test_render.py
```

### Модель данных (`model.py`)

- `Region(Enum)`: `WEST`, `EAST` (с человекочитаемой меткой и подписью таймзоны).
- `ShiftType(Enum)`: `DAY`, `NIGHT`, `OFF`, `VACATION` (со своим односимвольным глифом).
- `Analyst` (dataclass): `name: str`, `region: Region`, `allows_night: bool = False`,
  `offset: int = 0`, `vacation: set[int] = {}`, `day_off_requests: set[int] = {}`.
- `MonthSchedule` (dataclass): `year`, `month`, `n_days`, `grid: dict[str, list[ShiftType]]`.
  Методы: `working_count(day_index) -> int`, `region_working(day_index, region) -> int`,
  `shift_count(name) -> int`.

### Генерация (`generator.py`)

`generate(roster: list[Analyst], year, month) -> MonthSchedule` — проходы:

1. База 2/2 из `offset`.
2. Отпуск/запросы перекрывают (`VACATION` / `OFF`).
3. Ночные по покрытию (DAY→NIGHT при выполнении условий: флаг, рабочий день,
   предыдущий день выходной, тёмное окно региона не закрыто).

`inherit_offsets(roster, year, month) -> None` — сдвигает `offset` каждого аналитика на
`n_days % 4` и очищает разовые `vacation` / `day_off_requests`.

Детерминированность: при одинаковом входе — одинаковый выход (без random; либо с
фиксированным seed, если потребуется выбор кандидата).

### Валидатор (`validator.py`)

`validate(schedule, roster) -> list[Violation]`. `Violation` = `(severity, subject, message)`,
`severity ∈ {"hard", "soft"}`. Проверки:

Жёсткие:
- >2 рабочих (DAY/NIGHT) подряд у любого аналитика;
- отсутствие выходного между блоками смен;
- `NIGHT` без выходного перед ним;
- `NIGHT` у аналитика без `allows_night`;
- при чётной команде >2 — день, где покрытие региона < 2 (отпуск/Вх не считаются покрытием);
- запрос Вх не соблюдён (день не стал выходным);
- день отпуска (О) не соблюдён (день в `vacation` не стал `VACATION`);
- >3 рабочих дней подряд у аналитика с отпуском в этом месяце.

Мягкие:
- число смен аналитика вне диапазона 12..18;
- среднее по команде далеко от 15 (|avg-15| > 2).

### Вывод (`render.py`)

Чистая функция формирования строк (для тестируемости) + печать. Таблица по регионам:
строка-шапка с числами месяца и днями недели (ПВСЧПСВ), строки аналитиков с глифами,
ANSI-цвета по типу смены. Под таблицей — отчёт валидатора (сгруппирован hard/soft) и
сводка по числу смен. Поддержка `--no-color`.

### CLI (`cli.py`)

`python -m shift_scheduler.cli [--year Y] [--month M] [--months N] [--no-color]`.
По умолчанию — текущий месяц, встроенный демо-ростер. `--months N` показывает N месяцев
подряд, демонстрируя наследование фазы. **Первым делом** переключает stdout в UTF-8
(`sys.stdout.reconfigure(encoding="utf-8")`) — иначе кириллица/глифы падают на Windows cp1251.

## Демо-ростер

Встроенный детерминированный ростер (по мотивам исходного скриншота): 2 аналитика Запада,
4 Востока, у одного-двух `allows_night=True`, несколько `day_off_requests` (Вх) и хотя бы
один `vacation`, чтобы все правила были видны в выводе. Чётное число (6) — активирует
правило «мин. 2 в смене».

## Тестирование (TDD: RED → GREEN → REFACTOR)

- `test_inheritance.py`: три правила стыка месяцев на конкретных offset/длинах месяцев.
- `test_generator.py`: база 2/2; отпуск/запрос перекрывают; ночь только при флаге и с
  выходным перед ней; детерминированность.
- `test_validator.py`: каждый тип нарушения ловится на специально «сломанном» графике;
  чистый график не даёт жёстких нарушений.
- `test_render.py`: smoke — формирует строки без исключений, содержит шапку и глифы.

## Вне области (YAGNI)

- Веб/GUI-интерфейс.
- Чтение ростера из файла (только встроенный демо-ростер).
- Полноценный CSP/ILP-оптимизатор.
- Экспорт в внешние форматы.

## Известные ограничения

- Мягкое правило «среднее ~15» может нарушаться при обилии Вх/отпусков — это by design,
  отражается в отчёте, а не блокирует генерацию.
- «По потребности покрытия» для ночных реализуется эвристикой противофазы, а не точным
  расчётом часовых поясов.
